Residents On-Call Database

This database holds the meaning of life, the secrets of the universe and everything else. It's the number 42. You will spend large amounts of your life entering data into this software. We have attempted to make it as easy as possible.

One of the core assessment methods for competencies in Clinical Pathology is to create a log of clinical procedures/cases. This database was created to address the following issues:

  • Residents utilize a large number of medical literature resources in their decision-making
  • A knowledge base for answers to similar questions can increase efficiency
  • The benefits of previous efforts are lost without an archive
  • Residents are often outside the hospital while on call so a repository of info can be helpful
  • Having a database can improve resident performance and therefore patient safety

The proposed benefits of using this database include

  • Generation of legible, formatted reports of each call
  • The reports can be circulated to the pertinent laboratory directors even if the residents do not work with them directly
  • De-identified reports can be generated to serve as documentation of progress toward proficiency in clinical laboratory medicine
  • The database is an addendum to the patient's medical record, which is maintained intact in electronic format

Key objectives of the new database software

  • Minimize time spent using the software for data entry
  • Eliminate data loss
  • Minimize the impact of maintenance on IT staff
  • Mobile support

Accounts

Your Account

You can update your information at your account page. You can access your account page by clicking on your username at the right corner of the menu bar.

Users and user authentication levels in the system

A new user with network ID should be created on the database for that user to access the system.

There are 4 different types of accounts on this database

  • Read only users: They can only view the database, and also can comment on existing records.
  • Regular users: All user operations.
  • Power users: They can create new users, update user information, add new tags.
  • Administrators: Full read/write access on everything, including editing other users records.

Creating a new call

No fields are required

The system is designed to be as flexible as possible. Therefore no fields are required (other than a valid date and time, which is autopopulated). You may create a completely empty record.

Autosave

When editing a call, the contents of the page are saved every 30 seconds in case your browser crashes or something else happens to your computer. If you open up a new call on the same computer, you will presented with the choice to Restore the last autosaved page or to Ignore the last autosave and start a new page.

Formatting

As this database is read by others and is an extension of the patient's clinical record, please make an effort to format entries so they are easy to read.

Extra features

Hospital Number

If you enter a hospital number that already exists on the system,the form will automatically display other call entries added to the system for this patient.

Tags

Tags are intended to make call classification flexible but also straight-forward. Currently there is an emphasis on classifying calls by division so relevant faculty can quickly scan pertinent calls. You can only tag an entry with a tag that's previously defined. Take a look at the Tags page to see possible tags. To enter a tag, start typingits name, and either completely type the tag or select from autocomplete, then hit Enter. You can tag a call with as many tags as necessary.

Linking a call to other calls.

While editing Relevant Information, Action Taken or Follow Up, you can link to another call by typing the following: (call: 12345) (or use the link to Call button). The number you should enter is the Call Number. When you save the call and view it, the system will create a link to the call number 12345.

Linking to PUBMED

In a similar fashion to linking to other calls, while editing Relevant Information, Action Taken or Follow Up, you can create a link to a PUBMED article by typing following: (pubmed: 12345) (or use the PUBMED button). When you save the call and view it, the system will create a link to the PUBMED article number 12345.

Templates

Clicking the Templates dropdown button will show the starred templates. Clicking one of them will insert the template that's defined under that name to the text area you're editing.

You own your own templates. No one else can see them. Create as many templates as you wish.

Clicking More... will bring you the Template Manager (try it out).

New Template

To create a new template,

  1. Click on new button.
  2. Enter a template name, preferably a unique one to the field at right section at the top.
  3. Enter the template body, (remember this could be anything) to the big text area at right section.
  4. Click on save button.

Edit Template

To edit an existing template,

  1. Click on the template name from the left section. This will load up the template on right section.
  2. Change the template anyway you want, its name and its body.
  3. Click on save button.

Delete Template

To delete an existing template,

  1. Click on the template name from the left section. This will load up the template on right section.
  2. Click on delete button.

Star a Template

Starring a template brings this template to the Templates drop down menu. To star a template,

  1. Click on button next to the template name.
  2. To unstar, click on button again.

System wide templates

It is possible to make a template system-wide, so that everyone will see that template, but no one other than the creator can edit it.

Saving

Once you save the call, you will be taken to the Edit page, and the call will be live on the database.

Editing an existing call

See Creating a new Call section. Editing functions are same as creating functions.

Only you and the Administrator User can edit your call. Other people can comment on it (see Viewing).

All of the edits of a call are saved in the system. We keep all versions of all edits. It is potentially possible to recover them. If you want a call back, you need to buy a lot of doughnuts, and not just cheap ones, for everybody.

Deleting a Call

If you accidently entered a call more than once or for some other reason need to remove a call, you may delete it. No one will ask questions. Although, if you delete all of your calls, someone might.

In reality an call is never deleted. We tell everyone that it is deleted. But in reality it's not. See the above info box.

Viewing calls

From Edit page, List page, Home page you may view your or others calls.

There are some features in this view:

Flag this record

If you think that you need to follow up with this call, you may flag it. Only you can see and flag your own calls. It's only for you. Once you flag your call, it will appear on your home screen under a special tab, and also there will be a flag next to it on your home screen.

Like this record

You can "Like" any call you view that you find particularly helpful. Unlike Facebook, you may like it more than once. The rationale behind this is that you may come back to this record more than once throughout the years, and it may be helpful for you more than once.

Add a new comment

Anyone using the system can add a comment to any call record. You may delete and edit your own comments. When the call is printed, the comments are printed as well.

Print this record

Printed calls are not everything you see on the screen. Some screen elements are hidden. To print more than one call see Search.

Searching

Searching can be broken down into 4 different types of operations:

  • Where you search
    • Meta-data of the call record. This includes all of the fields other than Relevant Information, Action Taken or Follow Up
    • Body of the call record. This includes only the Relevant Information, Action Taken and Follow Up
    • All records. Naturally if you don't enter any keywords to the Search Query field, all results are returned.
  • How you search
    • Simple. Default mode is simple. Just enter each keyword on the search query separated by spaces. System will search and return the results that contain all of the search terms. No operators are allowed.
    • Advanced. If you select advanced query checkbox, the search query becomes a boolean search query. Boolean operations are AND, OR, NOT and you can group by parenthesis. For AND use & character, for OR use | character, for NOT use ! character. AND operation is not assumed by default (this means you can't do a query like term ! term2, you must type term & ! term2 which means search records that contain term but not term2.). You can use parenthesis ( and ) for grouping. Some examples are:
      • Search term and term2: term & term2
      • Search term and also either one of term2 or term3: term & (term2 | term3)
      • ((term & term2) | (term3 & term4) & ! term5) | term6 is possible, I can't explain what this means.
      Good luck!
  • How you limit the search
    • Of course the most obvious way to limit results are using a Search query
    • There are some fields that are on meta-data section that you can further limit your search results. We don't support multiple selections, sorry!
      • Hospital
      • Resident Name
      • Classification
      • Date Range: You don't have to fill both fields. The first field is from defining the lower bound, the second field is to defining the upper bound. So it's possible to search for all entries up to 2012, or all entries after March 4th.
      • Recent: A quick way to specify recent time span.
  • How you sort

    The default sorting tries to sort in a good way with respect to the search options you have selected. But you may overwrite the search results' sorting.

Printing search results

If you need to print multiple records, when search returns its results, there will be a Print these records button. Clicking will take you to a new page that renders all of the search results ready to be printed.

Listing calls

The List page lists all call entries in the system, sorted by their Call date and Call time.

Comments

If there are comments in a call, the column marked by shows the number of comments for that call.

Edit

If you are the owner for that call listed, there will be a edit link.

Printing

You may print calls several ways:

Printing a single call

While viewing the call, there is a Print this record button. Clicking that will bring print dialog. You may also just use your browsers Print option.

Printing multiple calls

The easiest way to print multiple calls is by doing a search.

I would like to print all of my call from my on-call week that happened over the last 4 days.

Go to Search, select your username from Residents, enter 4 and select "Days" from Recent section, select "By Id: smallest first" from Sort by section. Do the search, click on Print these records button.

You may bookmark a search result. Bookmarking a search result that states Recent 5 Days will dynamically return newest results that are matching last 5 days.

Home Screen

The Home screen consists of 3 tabs. All lists on this screen are limited to 100 results.

Latest calls

Your calls are sorted by the most recently updated first. This view allows you to quickly find and continue a call record you were working on.

Flagged calls

Once you flag a call at the Edit page, that call shows up in this tab. You can view / edit that record and remove the flag.

Commented calls

All calls that are commented by anyone are listed here, sorted by the most recent comment on top.

Tags

Power users can create new tags, and regular users (you!) can view them. Each tag has its own page that shows call records that are tagged with that tag. If you would like a new tag, let your Power User (chief resident) know.

Other features

Calls cannot be edited after some period

After 90 days, editing a call is locked. You can only comment on a call.

Stale data

Call records older than 3 years are considered stale. Viewing these records is allowed, but you will see a warning.

Bookmarking

You can bookmark, or share the URL of every page you see in this system.

Tour

Some pages contain a Tour feature to provide a quick overview - try it out!
